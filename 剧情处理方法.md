# 剧情文件处理方法

> 本文档记录将 LLM 生成的卡面剧情文本，整理为 noctchill 档案库标准格式的完整流程。
> 参考基准：[[tooru/]] 文件夹下的已完成文件。

---

## 一、文件结构标准

每份卡面分析文件的标准结构如下：

```markdown
# P-SSR【卡面名称】

[1-2句主题性介绍，说明这张卡在角色弧光中的位置与意义。]

---

## 剧情拆解：[副标题]

### 背景设定

[场景描述]

### 1. 核心台词：[小节主题]

> **角色：** 「日语台词」
>
> _（中文翻译）_

- **拆解：** ...
- **深意：** ...

### 2. 核心台词：[小节主题]

...

---

## 细节表现分析 (Direction Analysis)

- **[细节名]：** ...
- **[细节名]：** ...

---

## 总结：[总结标题]

[总结正文，无导引性问句，以有力的文学语言收束。]
```

---

## 二、处理三步骤（+ 额外步骤）

### Step 0 — 修复破损标题层级（若存在）

部分 LLM 输出存在 `### ##` 双前缀的格式错误（H3 + H2 混合），Obsidian 无法正确渲染。

**识别标志：** 章节标题以 `### ## ` 开头（三个井号 + 空格 + 两个井号）

**修复规则：** 删去多余的 `### `，统一改为 `## `：

| 原文（错误） | 修复后 |
|------------|--------|
| `### ## 剧情拆解：P-SSR【xxx】——` | `## 剧情拆解：xxx` |
| `### ## 细节表现分析 (Direction Analysis)` | `## 细节表现分析 (Direction Analysis)` |
| `### ## 总结：xxx` | `## 总结：xxx` |

> ⚠️ 此步骤仅在原文存在 `### ##` 时需要执行。早期批次（如 hnn 第一批）无此问题。

---

### Step 1 — 添加 H1 标题

LLM 生成的文本通常没有顶级标题，需在文件开头添加：

```markdown
# P-SSR【卡面名称】
```

> ⚠️ 注意：`主线剧情.md` 文件已自带 H1，无需此步。

---

### Step 2 — 精简开头段落

LLM 生成的文本开头通常包含**对话式语境段**（引用上一次对话内容）和**元叙述句**，需要删除或改写。

**识别并删除以下内容：**
- 以「既然我们已经……」「如果你觉得……」「看来你已经……」开头的语境衔接句
- 以「我们来拆解这场……」「接下来我们要……」结尾的过渡句

**保留并改写为客观陈述的内容：**
- 卡面标题的词源/文字游戏解析
- 卡面在角色弧光中的定位描述
- 与其他卡面的比较（删去"前一次对话"式引用，保留内容本身）

**改写示例：**

| 原文（删除） | 改写后（保留） |
|------------|--------------|
| 「既然上一张【♡AKUMA♡】让我们见识了……那么这张……」 | 「这张被称为"物理越界神卡"的 P-SSR【TRICK☆☆☆】……」 |
| 「如果你觉得前面已经够震撼，接下来……」 | 「被称为雏菜个人线"哲学最高山峰"的集大成之作……」 |
| 「我们来拆解这场关于……的终极对谈。」| （直接删除） |

---

### Step 3 — 清理结尾引导问句

LLM 生成的文本结尾通常有一段**导引问句**，引导用户继续对话，需完整删除。

**识别标志：**
- 独立段落，通常以 `**好了，……**` 或 `**接下来，你想……**` 开头
- 内容为"你想看 A 还是 B"的选择题形式
- 有时为两段：一段总结 + 一段问句，均需删除

**示例（删除整段）：**
```
**接下来，你想继续感受雏菜如何用"距离感"玩弄制作人的另一张神卡【TRICK☆☆☆】，
还是想转向 noctchill 组合里另外两位角色的剧情拆解呢？**
```

---

## 三、`## 剧情拆解：` 标题格式

LLM 生成的拆解标题通常包含卡面名称（因为没有 H1 做冗余），处理时需精简：

| 原格式 | 处理后 |
|--------|--------|
| `## 剧情拆解：P-SSR【♡AKUMA♡】—— 绝对支配的甜蜜陷阱` | `## 剧情拆解：绝对支配的甜蜜陷阱` |
| `## 剧情拆解：P-SSR【TRICK☆☆☆】—— 越界的特权` | `## 剧情拆解：越界的特权` |

规则：**删去 `P-SSR【卡面名】——` 部分，只保留副标题**。

---

## 四、处理检查清单

处理完一个文件后，对照以下清单验证：

- [ ] 文件第一行是 `# P-SSR【卡面名称】`
- [ ] 开头无「既然……」「看来……」「接下来我们要……」等对话式语境句
- [ ] 开头无「我们来拆解这场……」等元叙述过渡句
- [ ] `## 剧情拆解：` 后的标题中无重复卡面名（`P-SSR【xxx】——` 已删除）
- [ ] 文件末尾无 `**接下来……你想……还是……**` 等导引问句
- [ ] 总结段落以有力的文学语句收束，不以问句结尾
